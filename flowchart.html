Hermes Smart Travel: Itinerary Generation Flow

Step-by-Step Operational Flow (Client to Services)

1. User Initiates Search (Frontend)
User enters destination/dates into the **React/Vite** interface and clicks "Search."
↓
HTTP POST Request (Trip/Search)
2. API Request to Backend (Application Tier)
Frontend sends request with parameters and user token to the Backend API for validation.
↓
Authentication & Orchestration
3. Fetch Data & Integrate APIs (Data Tier)
Backend retrieves **User Preferences** (Database) and calls **External APIs** (Mapping, Booking, Weather) for real-time data.
↓
Business Logic & Aggregation
4. Process & Optimize Itinerary
Backend aggregates all data, applies business rules, and computes the final optimized itinerary proposal.
↓
HTTP Response (JSON Itinerary)
5. Render View Interface (Presentation Tier)
Frontend (React) receives the JSON, updates state, and renders the complete itinerary using responsive UI components.
↓
END: User Interacts with Results
The user can now view, save, or modify the generated trip.
